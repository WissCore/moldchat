## Becoming a maintainer

Maintainers are added by consensus of the existing maintainers based on sustained, high-quality contributions over time. There is no fixed criterion. Indicators we look at: code quality, review quality, security awareness, communication, alignment with the project's mission.

To express interest, open a GitHub Discussion explaining your background and the area you want to maintain.

## Maintainer responsibilities

- Review pull requests in your area within a reasonable time.
- Respond to security reports promptly.
- Uphold the [Code of Conduct](CODE_OF_CONDUCT.md).
- Use signed commits and a hardware-backed key for sensitive operations.
- Disclose conflicts of interest.

## Removing a maintainer

A maintainer may step down at any time by opening a PR removing themselves from this file. Removal for cause requires consensus of the other maintainers and is documented in [GOVERNANCE.md](GOVERNANCE.md).
